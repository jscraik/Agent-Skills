# Apps SDK: Research Use Cases (Condensed)

## Why
- Discovery in ChatGPT is model-driven; crisp use cases improve tool selection.

## Gather inputs
- Capture JTBD, user terminology, and data sources.
- Note compliance, offline data, and rate limits.

## Evaluation prompts
- 5 direct prompts, 5 indirect prompts, and negative prompts per use case.

## Minimum lovable feature
- What must be visible inline?
- Which actions require write access and confirmation?
- What state persists between turns?

## Tooling translation
- Define inputs/outputs and component intent per tool.
- Review with legal/compliance before implementation.
