# Spec Linter Checklist (Minimal)

## Foundation Spec

- Problem & Job is singular and clear.
- Success criteria include metric + target.
- Scope in/out is explicit.
- Primary journey is the happy path only.
- Evidence lines present per paragraph.
- Evidence Gaps + Evidence Map included.

## UX Spec

- All 6 passes are complete (Mental Model, IA, Affordances, Cognitive Load, State Design, Flow Integrity).
- IA covers entities + relationships + navigation.
- Affordances/actions mapped with explicit signals.
- Cognitive load friction points and defaults are listed.
- State tables include empty/loading/success/partial/error.
- Flow risks and visibility decisions are explicit.
- UX acceptance criteria are testable (Given/When/Then).
- State machine diagram included.
- Each component referenced in Pass 3 has a state machine diagram (or a note that it is stateless).
- Evidence lines present per paragraph.
- Evidence Gaps + Evidence Map included.

## Build Plan

- Outcome → Opportunities → Solution is explicit.
- Top risks/assumptions with mitigations.
- Epics are sequenced; stories are small with AC + validation.
- Test strategy includes unit/integration/E2E/failure.
- Release plan includes rollout + rollback + monitoring.
- Evidence lines present per paragraph.
- Evidence Gaps + Evidence Map included.
