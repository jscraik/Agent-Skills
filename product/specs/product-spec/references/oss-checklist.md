# OSS UX Checklist (Concise)

Use this checklist to review adoption and contributor experience.

## First Impression
- README explains what it is and who it is for.
- Screenshots or demo link are current.
- License is clear and prominent.

## Install and First Run
- Dependencies and versions are explicit.
- Quick start works as written.
- Common errors and fixes are documented.

## Onboarding and Docs
- Top tasks have clear docs or examples.
- Docs match the current UI/behavior.
- Troubleshooting is easy to find.

## Trust and Governance
- Security policy and disclosure path are visible.
- Release cadence and maintenance signals are clear.
- Contribution guide and code of conduct exist.

## UX Consistency
- UI behavior matches documentation.
- Defaults are safe and easy to undo.
- Error states preserve user inputs.
