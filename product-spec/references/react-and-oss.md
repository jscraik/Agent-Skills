# React and OSS Review Criteria (Concise)

Use this reference when the product is a React app or an open-source project.

## React UX Consistency
- Predictable component behavior across routes and states.
- Avoid layout shift during data loading; reserve space.
- Clear error boundaries and user-facing failure states.
- Accessibility: prefer semantic elements before ARIA.

## OSS Adoption Friction
- Installation and setup flow is obvious and low-effort.
- Clear version compatibility and requirements.
- First-run experience works with minimal configuration.
- Docs align with the current UI and behavior.

## Community and Trust
- Transparent licensing and data practices.
- Respect contributor workflows (issues, discussions, releases).
- Avoid dark patterns in signup, telemetry, or upgrades.
